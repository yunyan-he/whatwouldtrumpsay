# 项目全貌梳理：What Would Trump Say?

## 一、项目定位

*   **核心目标**：构建一个特朗普言论与行为的预测系统，基于历史数据预测他对当前事件的回应。
*   **产品形态**：每日推送大新闻 + AI预测特朗普回应 → 用户订阅关注 → 真实回应出现后自动对比验证 → 未覆盖事件补充进数据库形成闭环。
*   **发布方式**：开源 GitHub 项目，个人学习为主，不以盈利为直接目的，但有间接价值（作品集、技术积累、潜在学术价值）。

---

## 二、数据设计

### 数据起点与分层
1.  **人格基础层**：相对静态，包括从政之前的商业言论、媒体采访、《交易的艺术》等，体现他的性格模式和谈判逻辑，对所有时间段都有效。
2.  **政治立场层**：2015 年参选后逐渐系统化，按年份分段。
3.  **执政行为层**：第一任期、下野期、第二任期三个阶段，对同一议题态度会有明显漂移。

### 数据结构建议 (JSON)

```json
{
  "date": "2021-07-15",
  "event": "触发事件描述",
  "topic_tags": ["移民", "贸易", "中国"],
  "trump_response": "实际言论",
  "sentiment": "attack/support/deflect",
  "target": "Biden/China/media",
  "phase": "第一任期/下野期/第二任期"
}
```

### 时间感知验证（核心设计亮点）
测试集按时间隔离，预测 2021 年 7 月的回应时，只允许调用 2021 年 7 月之前的数据。这避免了数据泄露（Data Leakage），在学术上称为 **Walk-Forward Testing / Temporal Validation**。这是大多数同类工作没有做到的。

## 三、技术路线（三阶段递进）

1.  **第一阶段：RAG (Retrieval-Augmented Generation)**
    *   用 LangChain 实现，把所有历史言论存入向量数据库。
    *   新事件发生时检索最相似的历史情境，由大模型生成风格一致的回应。
    *   **特点**：成本低、上手快，是产品闭环的基础。

2.  **第二阶段：Fine-tuning**
    *   用言论数据微调语言模型，让模型学习他的语气、立场偏好、用词风格。
    *   **特点**：在 RAG 基础上叠加风格能力，但需要一定算力和数据清洗工作。

3.  **第三阶段：行为预测 Agent**
    *   用 LangGraph 实现，加入决策逻辑、利益关联、政治盟友关系等结构化特征。
    *   构建“看到新闻 → 判断议题 → 检索历史 → 生成预测 → 评估置信度”的多步骤推理流程。
    *   **特点**：预测他会采取什么行动，而非仅仅是言论。

三个阶段是递进叠加关系，先跑通第一阶段验证闭环，再逐步升级。

---

## 四、预测指标设计

早期只做两个可客观验证的维度：
1.  **立场预测**：支持/反对/攻击谁（分类任务），准确率可直接量化。
2.  **关键词预测**：他会用哪些标志性词汇。

后期升级加入：
*   **情绪强度**（发帖频率、用词激烈程度）。
*   **行动预测**（是否签行政令、发动贸易制裁等）。*注：最难，涉及内部决策。*

---

## 五、学术价值

有发表潜力，核心贡献点：
1.  **可验证预测闭环**：时间隔离的设计。
2.  **动态人格建模框架**：区分静态人格特征和动态立场演变。
3.  **技术栈创新**：LangChain + LangGraph 构建政治决策 Agent（Political NLP / Multi-Agent Systems）。
